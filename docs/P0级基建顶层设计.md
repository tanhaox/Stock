## 零、需求优先级权重

> **核心原则**（来自用户）：Regime → News → 庄家伪装 → 分时图 → **筹码位置（最基础）**
>
> 筹码位决定：1%盈利位 vs 90%盈利位，即使当日走势一致，次日表现完全不同

### 0.1 权重矩阵

| P0 | 权重 | 原因 | 实施优先级 |
|----|------|------|-----------|
| **P0-2 筹码** | **★★★★★** | 决定当日走势的本质含义，AlphaFlow 锁死信号核心来源 | **P0** |
| P0-1 除权 | ★★★★ | 前复权计算基础，无则所有历史分析失真 | P1 |
| P0-4 名称 | ★★★★ | 每笔分析必调，<1ms 是必须的 | P1 |
| P0-3 分时 | ★★★ | 形态识别辅助，非核心决策依据 | P2 |
| P0-6 资金 | ★★★ | 情绪辅助，次日表现的相关性低于筹码位 | P2 |
| P0-5 行业 | ★★ | 板块分析用，非单票决策核心 | P3 |

**实施顺序**: P0-2 → P0-1 → P0-4 → P0-3 → P0-6 → P0-5

### 2.1 存储策略决策

| 维度 | 评估 | 结论 |
|------|------|------|
| 调用频率 | L1 (>100次/日) | AlphaFlow/DeepScorer每个分析都调用 |
| 单次计算量 | 极低 (查询2条因子×乘法) | <1ms |

### 3.4 筹码计算公式（CVP 算法）

> **核心算法**: Cost Volume Positioning (CVP) — 成交量加权成本分布

```python
def compute_chip_distribution(
    klines: List[Kline],  # 至少需要 3 年日线
    levels: int = 100,     # 分档数
) -> ChipDistribution:
    """
    CVP 筹码分布计算:

    1. 构建价格区间
       price_min = min(klines.low)
       price_max = max(klines.high)
       step = (price_max - price_min) / levels

    2. 累积成交量到价格区间
       for k in klines:
           volume_at_price = k.volume * (k.high - k.low) / step
           for p in range(k.low, k.high, step):
               dist[int(p / step)] += volume_at_price

    3. 归一化
       total = sum(dist)
       dist = [d / total for d in dist]

    4. 计算关键指标
       - avg_cost: 加权平均价格 = Σ(price × volume) / Σ(volume)
       - median_cost: 中位成本（累积50%位置）
       - profit_ratio: 盈利筹码比例 = Σ(dist[price < current_price])
       - VIP_price: 筹码密集峰值位置

    5. AlphaFlow 锁死识别
       - compression_score = 筹码密集区宽度 / 历史宽度
       - lockup_days = 连续低于 VIP 且波动 < 5%
       - lockup_score = f(compression_score, lockup_days, volume_shrink)
    """

    # 关键公式
    avg_cost = sum(p * v for p, v in zip(prices, volumes)) / sum(volumes)
    median_cost = find_cumulative_level(dist, 0.5)
    profit_ratio = sum(dist[i] for i, p in enumerate(prices) if p < current_price)
    vip_ratio = current_price / vip_price  # >1=在VIP上方，<1=在VIP下方
```

> - 按需从 Tushare 拉取，实时计算
> - LRU 缓存近期数据

```python
class MinuteService:

    # 策略: L2 分级存储，聚合优先

    async def detect_patterns(

    # 采集层（Tushare stk_mins）
    async def fetch(ts_code: str, date: date, freq: str = '1min') -> int:

        策略:
        - 历史: 按月拉取1分钟，存储到 minute_kline_raw
        - 当日: 拉取1分钟 → 聚合5/15/30/60分钟
        - 限制: 每日最多拉取N只（积分限制）
        """

    # 聚合计算（每日收盘后）
    async def aggregate(date: date, periods: List[str] = ['5min','15min','30min','60min']):
        """
        从 minute_kline_raw 读取当日1分钟
        聚合到 minute_agg
        识别形态到 minute_patterns
        耗时: ~10分钟全市场
        """

    # 相似走势搜索（高级功能）
    async def find_similar(
        pattern: List[float],
        ts_code: str = None,  # None=跨股票搜索
        period: str = '15min',
        threshold: float = 0.8
    ) -> List[SimilarMatch]:
        """
        使用向量相似度（cosine similarity）
        从 minute_patterns.features 搜索
        """
```

```
采集优先级（积分限制200次/分钟）:
1. 每日收盘后: 拉取当日1分钟 → 聚合 → 存储
2. 批量回溯: 按股票×按月拉取（避免超限）
3. 热点股: 保留近20日1分钟原始
4. 其他: 只保留聚合周期

积分消耗估算:
- 每日采集: 5000股票 ÷ 200次/分 = 25分钟
```

## 十、核心决策总结

```
存储策略决策:
├── 数据量大 + 计算量高  → 预计算快照本地存
├── 数据量小 + 调用频繁  → 全量本地 + 内存缓存
├── 数据量中等           → 全量本地 + 定期同步
└── 数据量极大           → 分级存储 + 按需拉取

Tushare调用策略:
├── 高频查询  → 本地优先，失败回Tushare
├── 低频查询  → Tushare优先，结果缓存
├── 无此接口  → 自研计算
└── 批量同步  → 定时任务，积分限制内
```

## 附录E：核心公式速查

### E.1 复权计算

```
前复权价格 = 当日价格 × adj_factor_当日 / adj_factor_历史
后复权价格 = 当日价格 × adj_factor_当日 / adj_factor_历史 (乘以不同基准)

例: 某股票 2020-01-01 价格 10元，adj_factor=2.0
    2024-06-01 价格 50元，adj_factor=1.0
    2020-01-01 前复权价格 = 10 × 1.0 / 2.0 = 5元
```

### E.2 筹码分布

```
平均成本 = Σ(价格 × 成交量) / Σ(成交量)
盈利筹码比例 = Σ(成交量[价格 < 当前价格]) / Σ(总成交量)
VIP 比值 = 当前价格 / 筹码密集峰值价格 (>1=低估，<1=高估)
```

### E.3 分时聚合

```
5分钟聚合:
  open = 第一条1min.open
  close = 最后一条1min.close
  high = max(所有1min.high)
  low = min(所有1min.low)
  volume = sum(所有1min.volume)
  amount = sum(所有1min.amount)
```
