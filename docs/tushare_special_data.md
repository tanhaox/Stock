# Tushare 特色数据调研 & 接入计划

> **调研日期**: 2026-06-09 | **实测结果**: 已在 8000 积分环境下逐接口实测

---

## 一、已实测可用的特色数据接口

| 接口 | 实测数据 | 字段 | 本系统用途 |
|------|---------|------|-----------|
| `cyq_perf` | ✅ 2043 rows (000001.SZ) | his_low/high, cost_5/15/50/85/95pct, weight_avg, winner_rate | 真实筹码成本五分位+获利盘比例 |
| `cyq_chips` | ✅ 6000 rows, 58天 (000001.SZ) | price, percent (105档/天) | 完整筹码密度曲线 |
| `cyq_perf` 纯不传日期 | ✅ 2043 rows | 同上，返回全量历史 | 筹码成本变化趋势追踪 |

### cyq_perf 数据字段详解

```
{
  "ts_code": "000001.SZ",
  "trade_date": "20260608",
  "his_low": 0.2,        # 历史最低价
  "his_high": 21.0,      # 历史最高价
  "cost_5pct": 10.4,     # P5: 5%筹码成本 <= 10.4元 (最便宜那5%的人的买入价)
  "cost_15pct": 10.8,    # P15: 15%筹码成本 <= 10.8元
  "cost_50pct": 11.2,    # P50: 筹码中位成本 (50%的人成本在此之下)
  "cost_85pct": 11.6,    # P85: 85%筹码成本 <= 11.6元
  "cost_95pct": 12.6,    # P95: 95%筹码成本 <= 12.6元 (高成本套牢盘的边界)
  "weight_avg": 11.33,   # 加权平均成本
  "winner_rate": 48.87   # 获利盘比例% (当前价高于平均成本的比例)
}
```

### cyq_chips 数据字段详解

```
{
  "ts_code": "000001.SZ",
  "trade_date": "20260608",
  "price": 21.0,         # 价格档位
  "percent": 0.1          # 该档位持仓筹码占总股本的比例%
}
# 每日 105 个档位 (his_low → his_high, 0.2元/档)
# → 可构建完整的筹码-价格分布直方图
```

### 与本系统现有手算筹码的关系

| 能力 | 现有手算 (chip_analyzer) | Tushare cyq_* | 差距 |
|------|------------------------|---------------|------|
| 筹码成本五分位 | ❌ 无 | ✅ cyq_perf | 巨大 |
| 获利盘比例 | ❌ 无 | ✅ winner_rate | 巨大 |
| 筹码密度曲线 | ❌ 无（只有三区量价近似） | ✅ cyq_chips 105档 | 巨大 |
| 锁死区吸收率 | ✅ AR_lock 三区模型 | — | cyq不提供（需本地算） |
| 三区量价归因 | ✅ Z_LOCK/Z_OVER/Z_BELOW | — | cyq不提供 |
| 数据源 | 5分钟线 + 日线 | Tushare 服务端逐笔聚合 | cyq精度碾压 |

### 退役策略

- `chip_analyzer.py` 的 `analyze_chip_absorption()` 签名保留，内部数据源从 5分钟线→cyq_perf
- `define_zones()` + `compute_absorption()` 的 **5分钟线手算逻辑退役**，保留三区框架
- 接口兼容：调用方无需改代码，返回字段不变，新增 cyq 字段透出
